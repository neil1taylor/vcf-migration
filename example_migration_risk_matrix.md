# Comprehensive Migration Risk Table: VMware to IBM Cloud VPC

| Category | Risk Description | Impact Area | Status | Mitigation Plan (Next Steps) |
| :--- | :--- | :--- | :---: | :--- |
| **Financial** | **Double Spend / Overlap Run-Costs:** Paying for existing bare-metal VMware hosts *and* new VPC instances simultaneously if the migration stretches out. | Budget | 🔴 | Plan tightly coupled "Move Groups" to quickly evacuate and cancel entire bare-metal servers. |
| **Financial / Licensing** | **OS & App Licensing Invalidation:** Moving from "Bring Your Own License" (BYOL) on vSphere to VPC. 3rd-party apps may not be licensed for non-VMware cloud environments. | Compliance / Legal | 🟡 | Audit all Windows/RHEL and ISV application licenses for cloud portability before migrating. |
| **Business / Org** | **Business Operations Disruption:** Cutover takes longer than the approved weekend maintenance window, impacting Monday business. | Revenue / Uptime | 🔴 | Establish strict "Point of No Return" timelines for rollback during migration weekends. Mandate dry-runs for Tier-1 apps. |
| **Business / Org** | **Migration Fatigue:** The core IT team is expected to handle their "day jobs" while also executing a massive, high-stress cloud migration project. | Schedule / HR | 🟡 | Backfill operational roles with contractors, or pause non-critical internal IT roadmap projects to free up capacity. |
| **Skills / Knowledge** | **Security Paradigm Shift:** Team is used to edge firewalls and NSX, but VPC uses Security Groups and NACLs. Misunderstanding this exposes applications. | Security | 🔴 | Conduct mandatory hands-on enablement for the network/security teams specifically on IBM Cloud VPC networking architecture. |
| **Skills / Knowledge** | **Cloud Operating Model Gap:** Team lacks the Infrastructure-as-Code (Terraform/Ansible) skills needed to manage VPC efficiently at scale. | Operations | 🟡 | Pair internal staff with cloud consultants for the first 3 migration waves. Mandate Terraform training. |
| **Ops & Tooling** | **Loss of Legacy Automation:** Custom vRealize, PowerCLI, or vCenter templates are useless in the new VPC environment. | Provisioning Time | 🔴 | Treat "Tooling Migration" as Phase 0. Rewrite deployment pipelines using IBM Cloud Schematics and Ansible. |
| **Ops & Tooling** | **Monitoring Blindspots:** vROps and VMware-specific dashboards will no longer work, leading to visibility gaps into application health. | MTTR (Recovery) | 🟡 | Deploy and map IBM Cloud Monitoring (Sysdig) agents to all workloads prior to final cutover. |
| **Ops & Tooling** | **Change Management / ITSM Breakage:** ServiceNow/Jira workflows designed around VMware VM provisioning no longer align with cloud approvals. | Governance | 🟢 | Map the new VPC provisioning workflow and update the ITSM tool configurations. |
| **Backup & DR** | **Incompatible Backup Architecture:** Tools like Veeam or Zerto (common in VMware) may not natively back up VPC instances the same way. | Data Protection | 🔴 | Design and implement IBM Cloud Backup for VPC policies; test block-level restores before moving production data. |
| **Backup & DR** | **Failure to meet RPO/RTO:** The previous cross-site VMware replication is gone. The new architecture might require multi-zone/multi-region VPC design to meet SLAs. | Business Continuity | 🔴 | Redesign the DR strategy using IBM Cloud cross-region replication and load balancers. |
| **Technical** | **Image Conversion / Boot Failure:** Exporting a VMDK and importing it as a VPC Custom Image fails because `cloud-init` or `virtio` drivers aren't present. Windows Server 2003 has no viable VirtIO drivers. Windows Server 2008/2008 R2 drivers dropped from current virtio-win releases (last working: 0.1.173.9); IBM Cloud requires virtio-win 1.9.24+ which excludes 2008. | Schedule | 🟡 | For Windows Server 2012 R2+: standardize a pre-migration script to inject current virtio-win drivers. For 2008 R2: archived virtio-win-0.1.173.9 may work but is unsupported — OS upgrade to 2019/2022 strongly recommended. For 2003: no viable drivers — OS upgrade or re-platforming required. |
| **Technical** | **Classic-to-VPC Routing Drops:** Overlapping IP spaces or misconfigured IBM Transit Gateway drops traffic between the migrated and un-migrated VMs. | Application Uptime | 🟡 | Map all IP subnets rigorously. Test Transit Gateway connectivity with dummy workloads first. |
